## 任务描述

在 `apps/app/src/index.tsx` 中注册 `/mode-select` 与 `/cockpit` 两条具名路由，并创建对应页面组件的骨架文件（空页面，可路由到达，返回占位内容），为后续 TASK 提供可工作的路由基础。

本 TASK 不实现任何业务逻辑，仅建立文件骨架和路由连接。

### 包含

- 在 `index.tsx` 的 `RouterComponent` 中，于 `<Route path="*all">` 之前新增两条路由
- 创建 `apps/app/src/app/pages/mode-select.tsx`：导出 `ModeSelectPage` 组件，返回可见的占位 `<div>`
- 创建 `apps/app/src/app/pages/cockpit.tsx`：导出 `CockpitPage` 组件，返回可见的占位 `<div>`（含 Tab 占位注释）
- 导出 `TabId` 类型定义于 `cockpit.tsx`

### 不包含（由其他 TASK 负责）

- ModeSelectPage 业务逻辑（由 TASK-002-02 负责）
- CockpitPage 布局与 Tab 逻辑（由 TASK-002-03 负责）
- AppEntry 自动跳转逻辑（由 TASK-002-02 负责）

## 实现要点

- [ ] `apps/app/src/index.tsx` — 注册双路由
  - 在现有 `<Route path="*all" component={() => null} />` **之前**插入：
    ```tsx
    import { lazy } from "solid-js";
    const ModeSelectPage = lazy(() => import("./app/pages/mode-select"));
    const CockpitPage    = lazy(() => import("./app/pages/cockpit"));
    // ...
    <Route path="/mode-select" component={ModeSelectPage} />
    <Route path="/cockpit"     component={CockpitPage} />
    ```
  - 保持 `*all` 通配路由不变，确保现有 openwork 体验不受影响
  - 使用 `lazy()` 包裹以支持代码分割，降低初始包体积

- [ ] `apps/app/src/app/pages/mode-select.tsx` — ModeSelectPage 骨架
  - 导出默认函数组件 `ModeSelectPage`
  - 返回带 `data-testid="mode-select-page"` 属性的容器 `<div>`，含占位文字"Mode Select"
  - 文件顶部添加注释：`// 业务逻辑由 TASK-002-02 实现`

- [ ] `apps/app/src/app/pages/cockpit.tsx` — CockpitPage 骨架 + TabId 类型
  - 在文件顶部声明并导出 `TabId` 类型：
    ```typescript
    export type TabId = "product" | "engineering" | "release" | "growth";
    ```
  - 导出默认函数组件 `CockpitPage`
  - 返回带 `data-testid="cockpit-page"` 属性的容器 `<div>`，含 Tab 占位注释
  - 文件顶部添加注释：`// 布局与 Tab 逻辑由 TASK-002-03 实现`

- [ ] 本地验证
  - `pnpm dev` 启动后，手动访问 `/#/mode-select` 可看到占位页面
  - 手动访问 `/#/cockpit` 可看到占位页面
  - 访问 `/#/`（根路由）行为与改动前完全一致

## 备注

- SolidJS `lazy()` 需从 `"solid-js"` 导入，非 React 的 `React.lazy`
- 路由注册顺序敏感：具名路由必须在 `*all` 通配之前，否则永远命中通配
- `HashRouter` 模式下，URL 格式为 `/#/mode-select`，本地开发时注意地址栏格式
- 参考 SDD-002 §3.2「路由注册变更」代码示例
