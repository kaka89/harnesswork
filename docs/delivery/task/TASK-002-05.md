## 任务描述

实现工程驾驶舱研发 Tab 容器组件 `EngineeringTab`，通过 SolidJS `lazy()` 懒加载嵌入现有 `SessionView`（`apps/app/src/app/pages/session.tsx`），并处理 Tab 内容区高度约束，避免双滚动条问题。

设计核心：最小化代码重复，不重写 SessionView 功能，仅处理容器尺寸适配。

### 包含

- EngineeringTab 容器组件（`h-full` 容器，撑满 CockpitPage 内容区）
- `lazy(() => import("../pages/session"))` 懒加载 SessionView，避免初始 bundle 过大
- `<Suspense>` 包裹 lazy SessionView，显示骨架屏等待加载
- 可选：透传 `workspaceId?: string` prop 给 SessionView（MODULE-002 EngineeringTab Props 定义）
- 修改 cockpit.tsx 研发 Tab 占位替换为 `<EngineeringTab />`

### 不包含（由其他 TASK 负责）

- SessionView 内部实现（现有代码，不修改）
- CockpitPage 布局框架（由 TASK-002-03 负责）

## 实现要点

- [ ] `apps/app/src/app/components/cockpit/engineering-tab.tsx` — EngineeringTab 组件
  - 使用 `lazy()` 延迟导入 SessionView：
    ```typescript
    import { lazy, Suspense } from "solid-js";
    const SessionView = lazy(() => import("../../pages/session"));
    ```
  - Props 接口（参考 MODULE-002）：
    ```typescript
    interface EngineeringTabProps {
      workspaceId?: string;
    }
    ```
  - 高度约束处理（Q2 方案）：
    ```tsx
    <div class="h-full overflow-hidden" data-testid="engineering-tab">
      <Suspense fallback={<EngineeringTabSkeleton />}>
        <SessionView workspaceId={props.workspaceId} />
      </Suspense>
    </div>
    ```
  - EngineeringTabSkeleton：3 栏骨架屏，左/中/右比例约 20%/55%/25%，使用灰色 `animate-pulse` 占位
  - 若 SessionView 不接受 workspaceId prop，则不传入，以不破坏现有组件签名为准

- [ ] 调查 SessionView 的高度约束行为（Q2）
  - 查看 `apps/app/src/app/pages/session.tsx` 顶层容器是否使用 `h-screen` 或 `min-h-screen`
  - 若使用 `h-screen`，需在 EngineeringTab 外层覆盖为相对高度：
    ```tsx
    <div class="h-full [&>*]:!h-full overflow-hidden">
    ```
  - 若无法通过 CSS 覆盖，评估是否需要向 SessionView 添加 `className` prop（范围最小变更）
  - 在 PR 描述中记录最终 Q2 方案结论

- [ ] 修改 `apps/app/src/app/pages/cockpit.tsx`
  - 导入 `EngineeringTab`（lazy 加载或直接导入）
  - 将研发 Tab 占位替换：
    ```tsx
    <Match when={activeTab() === "engineering"}>
      <EngineeringTab />
    </Match>
    ```

- [ ] 性能评估（R1 风险项）
  - 首次切换到研发 Tab 时，记录控制台时间戳，评估 SessionView 懒加载耗时
  - 若超过 500ms（含网络+解析），在 PR 描述中记录，供 tech-lead 决策是否预加载
