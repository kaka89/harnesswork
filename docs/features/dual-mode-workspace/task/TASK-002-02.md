## 任务描述

完整实现 `ModeSelectPage` 组件，包含视觉区分的两模式选项按钮、LocalStorage 模式偏好读写、点击后路由跳转。同时在 `AppEntry` 组件挂载时读取 `harnesswork:mode-preference`，实现自动跳转逻辑。

### 包含

- ModeSelectPage 完整 UI：居中布局，两个视觉区分的模式卡片/按钮
- 点击"openwork 原始版本" → 写入 `openwork` → navigate `"/"`
- 点击"harnesswork 工程驾驶舱" → 写入 `cockpit` → navigate `"/cockpit"`
- `platform.storage("harnesswork")` 读写封装（getItem / setItem）
- AppEntry 挂载时自动跳转：偏好为 `cockpit` + 当前路由为 `/` → navigate `"/cockpit"`
- 驾驶舱顶部"返回模式选择"按钮**仅**在本文件中定义跳转目标（`"/mode-select"`），实际按钮由 TASK-002-03 在 CockpitPage 中实现

### 不包含（由其他 TASK 负责）

- CockpitPage 布局（由 TASK-002-03 负责）
- 驾驶舱顶部返回按钮的渲染（由 TASK-002-03 负责）

## 实现要点

- [ ] `apps/app/src/app/pages/mode-select.tsx` — 完整实现 ModeSelectPage
  - 整体布局：`<div class="flex items-center justify-center min-h-screen bg-gray-950">` 全屏居中
  - 内容区：两张模式卡片并排（`flex gap-8`），每张卡片包含图标、标题、描述、点击区域
  - "openwork 原始版本"卡片：
    - 标题："openwork 原始版本"，副标题："AI 编码工作流"
    - 点击处理：`() => { storage.setItem("mode-preference", "openwork"); navigate("/") }`
    - 样式：`border border-gray-700 hover:border-gray-400 cursor-pointer`
  - "harnesswork 工程驾驶舱"卡片：
    - 标题："harnesswork 工程驾驶舱"，副标题："全链路研发协作"
    - 点击处理：`() => { storage.setItem("mode-preference", "cockpit"); navigate("/cockpit") }`
    - 样式：`border border-blue-600 hover:border-blue-400 cursor-pointer`（视觉强调）
  - 使用 `useNavigate` 来自 `@solidjs/router`
  - `data-testid="mode-select-page"` 保留在根容器

- [ ] `platform.storage` 封装调用
  - 在组件内或单独 helper 文件中封装：
    ```typescript
    const storage = platform.storage("harnesswork");
    // getItem("mode-preference") → "openwork" | "cockpit" | null
    // setItem("mode-preference", value)
    ```
  - 若 `platform.storage` 为异步接口（Q1 待确认），用 `createResource` 包裹，显示加载态后再渲染两张卡片
  - 若为同步接口，直接调用

- [ ] `apps/app/src/app/entry.tsx` — AppEntry 自动跳转逻辑
  - 在 `onMount`（或 `createEffect`）中执行：
    ```typescript
    const pref = platform.storage("harnesswork").getItem("mode-preference");
    const location = useLocation();
    if (pref === "cockpit" && location.pathname === "/") {
      navigate("/cockpit");
    }
    ```
  - 仅在当前路径为 `/` 时触发，避免访问 `/mode-select` 时被强制跳转
  - 不修改 AppEntry 其他渲染逻辑

- [ ] 视觉验收
  - 两张卡片等宽、等高，视觉上有明确区分（颜色、边框或背景差异）
  - hover 状态有反馈
  - 文字清晰可读，在深色背景下对比度满足 WCAG AA
