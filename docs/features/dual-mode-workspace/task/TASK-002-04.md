## 任务描述

实现工程驾驶舱产品 Tab 的完整功能：左侧 DocTreePanel（调用 GET /docs 渲染文档层级树，支持展开/折叠、状态标签）和右侧 DocViewerPanel（调用 GET /docs/:path 获取并渲染 Markdown 内容）。两者通过 ProductTab 容器组件左右分栏布局组合。

**前置条件**：TASK-002-03（CockpitPage 容器）和 TASK-002-06（Server 端点）均已合入。

### 包含

- ProductTab 容器：左右分栏布局（左侧 DocTreePanel 固定宽度 + 右侧 DocViewerPanel 撑满）
- DocTreePanel：挂载时调用 GET /docs，按路径前缀分组渲染 PRD/SDD/PLAN/TASK/MODULE 层级树，各节点显示状态标签（draft/approved/released），支持展开/折叠
- DocViewerPanel：接收 path prop，调用 GET /docs/:path 获取 Markdown，渲染展示；处理 404、403/500 错误状态
- 将 cockpit.tsx 中产品 Tab 占位替换为 `<ProductTab />`

### 不包含（由其他 TASK 负责）

- GET /docs 和 GET /docs/:path 服务端实现（由 TASK-002-06 负责）
- CockpitPage 布局框架（由 TASK-002-03 负责）

## 实现要点

- [x] `apps/app/src/app/components/cockpit/doc-tree-panel.tsx` — 文档树面板
  - Props：`interface DocTreePanelProps { onSelect: (path: string) => void }`
  - 挂载时用 `createResource` 调用 `fetch("/docs")` 拉取 `DocEntry[]`
  - 将 DocEntry[] 按路径第一级前缀分组（如 `product/prd/` → "PRD"，`product/architecture/` → "SDD" 等）
  - 每个文档节点渲染：文件名（title）+ 状态标签 badge（`draft` 灰色，`approved` 绿色，`released` 蓝色）
  - 点击叶子节点调用 `onSelect(entry.path)`
  - 支持展开/折叠：用 `createSignal<Set<string>>` 管理展开状态
  - 空状态（BH-18）：GET /docs 返回 `[]` → 显示 `<p data-testid="doc-tree-empty">暂无文档</p>`
  - 加载状态：`createResource` pending 时显示骨架屏（3 个灰色占位条）
  - `data-testid="doc-tree-panel"` 在根容器

- [x] `apps/app/src/app/components/cockpit/doc-viewer-panel.tsx` — Markdown 渲染面板
  - Props：`interface DocViewerPanelProps { path: string }`
  - 用 `createResource(() => props.path, fetchDocContent)` 在 path 变化时自动重新请求
  - `fetchDocContent(path)` → `fetch(\`/docs/\${encodeURIComponent(path)}\`)`，返回 Response.text()
  - 成功（BH-19）：将 Markdown 文本转换为 HTML 渲染（选型：Q4，优先复用现有 Markdown 渲染工具；若无，引入 `marked` 并通过 `innerHTML`（须 sanitize）或 `solid-markdown` 渲染）
  - 404（BH-20）：显示 `<p data-testid="doc-viewer-not-found">文档未找到</p>`
  - 403/500（BH-21）：显示 `<p data-testid="doc-viewer-error">加载失败，请重试</p>`
  - 初始 path 为空时：显示"请选择左侧文档"引导提示
  - `data-testid="doc-viewer-panel"` 在根容器

- [x] `apps/app/src/app/components/cockpit/product-tab.tsx` — ProductTab 容器
  - 内部 Signal：`const [selectedPath, setSelectedPath] = createSignal("")`
  - 布局：
    ```tsx
    <div class="flex h-full" data-testid="product-tab">
      <aside class="w-64 shrink-0 border-r border-gray-800 overflow-y-auto">
        <DocTreePanel onSelect={setSelectedPath} />
      </aside>
      <section class="flex-1 overflow-y-auto p-4">
        <DocViewerPanel path={selectedPath()} />
      </section>
    </div>
    ```

- [x] 修改 `apps/app/src/app/pages/cockpit.tsx`
  - 导入 `ProductTab`（lazy 加载或直接导入）
  - 将 `<Match when={activeTab() === "product"}><div data-testid="tab-product-placeholder" /></Match>` 替换为 `<ProductTab />`
