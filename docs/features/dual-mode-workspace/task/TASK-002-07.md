## 任务描述

实现工程驾驶舱发布&运维 Tab 的 P1 静态 Mock 版本（`ReleaseTab`）。使用 `IS_MOCK = true` 常量控制数据来源，渲染四个面板：流水线执行视图、部署历史时间轴、多环境健康看板、告警摘要面板。

P1 阶段全部使用 Mock JSON 数据，不接入任何真实 CI/CD API。P2 阶段通过 Feature Flag 切换，另立 SDD 描述后端集成方案。

### 包含

- `IS_MOCK` 常量定义（`const IS_MOCK = true`），置于文件顶部
- 四个 Mock 数据面板渲染：
  1. 流水线执行视图（Pipeline）：展示 2~3 条 Mock 流水线，含阶段/状态指示
  2. 部署历史时间轴（Deploy History）：展示 3~5 条 Mock 部署记录
  3. 多环境健康看板（Environment Health）：dev/staging/prod 三环境健康状态
  4. 告警摘要面板（Alerts）：2~3 条 Mock 告警，含级别（critical/warning/info）
- 将 cockpit.tsx 发布&运维 Tab 占位替换为 `<ReleaseTab />`
- Mock 数据标注"数据仅供演示"提示

### 不包含（由其他 TASK 负责）

- 真实 CI/CD API 接入（P2，另立 SDD）
- 快捷操作（审批/回滚）的实际功能（P2）
- CockpitPage 布局框架（由 TASK-002-03 负责）

## 备注

- ReleaseTab 参考 Spinnaker / Harness CD / Argo CD 的设计风格（见 PRD-002 FR-06 描述），P1 仅还原视觉框架
- `IS_MOCK` 常量命名规范：全大写，置于文件顶部，便于 P2 阶段全局搜索替换
- P2 阶段接入真实 CI/CD API 时将另立 SDD，`IS_MOCK` 切换为 `false` 并实现 `fetchPipelines()` 等函数
- Mock 数据应具备代表性（包含 success/running/failed 三种状态），以便 UI 验收时覆盖所有视觉分支
