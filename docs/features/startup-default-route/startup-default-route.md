## 特性描述

当前 harnesswork 启动后默认进入 openwork 原有首页（`/`），用户无法直接看到模式选择入口。本特性将应用冷启动的默认路由从 `/` 改为 `/mode-select`，确保用户每次启动都能主动选择进入哪种工作模式。同时调整 FR-08 的 LocalStorage 模式记忆行为，改为「高亮上次所选」而非「自动跳过选择页直接进入」，保持用户对启动模式的主动控制权。

## 关键决策

### 默认路由入口变更策略

- **决策**：将 `apps/app/src/main.tsx`（或路由配置文件）中的默认路由从 `/` 改为 `/mode-select`
- **备选方案 A**：保持 `/` 为默认，在 `/` 挂载一个重定向逻辑检测 localStorage 决定跳转
- **备选方案 B**：完全替换 `/`，让 ModeSelectPage 接管 `/` 路由
- **选择理由**：保留 `/` 作为 openwork 原始首页路由不变，仅将启动默认入口切换为 `/mode-select`，避免破坏 openwork 原有路由结构，降低兼容风险
- **代价**：需要在路由守卫或 App 初始化逻辑中添加重定向，而非直接修改根路由

### LocalStorage 模式记忆行为调整

- **决策**：FR-08 行为从「下次启动自动进入上次模式」改为「高亮上次所选，但始终展示选择页」
- **备选方案**：保留自动跳转，增加「每次都问我」开关
- **选择理由**：用户明确希望保持主动选择权，自动跳转会让用户不知道有模式切换入口，增加认知负担
- **代价**：用户每次启动都需要点击一次确认，略微增加操作步骤

## 行为规格一览

| 编号 | 场景 | 预期 |
|------|------|------|
| BH-01 | 用户首次启动 harnesswork | 默认显示 `/mode-select` 页，无高亮预选 |
| BH-02 | 用户选择过一次模式后再次启动 | 显示 `/mode-select` 页，上次选择的选项高亮 |
| BH-03 | 用户直接访问 `/` 路由 | 正常显示 openwork 原始首页（不受影响） |
| BH-04 | 用户在驾驶舱内点击「返回模式选择」 | 跳回 `/mode-select`（已有功能，不受本特性影响） |
| BH-05 | localStorage 数据被清除后启动 | 回退到 BH-01 无高亮状态 |
