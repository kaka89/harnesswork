## 1. 背景

harnesswork 基于 openwork fork 构建，在继承 openwork 原始 AI 编码体验的同时，正在扩展为一个面向完整产品研发生命周期的工程驾驶舱（Engineering Cockpit）。当前版本缺乏入口路由机制——用户无法在"使用 openwork 原始工作流"和"进入 harnesswork 扩展工作区"之间做出明确选择。

同时，harnesswork 的改造版本需要提供一个统一视图，覆盖产品研发全链路节点（产品 / 研发 / 发布&运维 / 运营），让团队中不同角色的成员都能在同一个工具内完成对应的工作。

## 2. 目标用户

| 用户角色 | 描述 | 核心诉求 |
|---------|------|---------|
| 工程师（Bob） | AI 辅助编码的主要用户 | 快速进入熟悉的 AI 编码工作流，或切换到工程驾驶舱视图 |
| 产品经理（PM） | 负责需求管理和产品文档 | 在统一界面内管理 PRD/SDD 等产品文档，全流程可见 |
| DevOps / SRE | 负责发布和运维管理 | 在统一界面内查看部署状态、告警和运维操作 |
| 运营人员 | 负责产品运营数据分析 | 在统一界面内查看运营指标、用户反馈和增长数据 |

## 3. 功能说明

### 3.1 用户故事

**模式选择入口**
> As a 用户,
> I want to 在启动后看到清晰的模式选择界面,
> So that 可以根据当前工作场景快速进入 openwork 原始模式或 harnesswork 工程驾驶舱模式，避免界面混乱。

**工程驾驶舱——产品节点**
> As a 产品经理,
> I want to 在产品 Tab 下浏览和管理结构化的产品文档（PRD/SDD/PLAN/TASK）,
> So that 无需切换工具即可完成从需求到交付状态的全程跟踪。

**工程驾驶舱——研发节点**
> As a 工程师,
> I want to 在研发 Tab 下看到代码树、编辑区和 AI 对话三栏布局，
> So that 在一个界面内完成代码阅读、编辑和 AI 辅助，与 Qoder 类似的高效研发体验。

**工程驾驶舱——发布&运维节点**
> As a DevOps 工程师,
> I want to 在发布&运维 Tab 下看到流水线状态、部署历史、环境健康度和告警面板,
> So that 快速掌握发布进度和线上状态，支持一键审批、回滚和事故响应。

**工程驾驶舱——运营节点**
> As a 运营人员,
> I want to 在运营 Tab 下查看用户增长、留存漏斗、关键事件和用户反馈聚合,
> So that 数据驱动地发现产品问题和增长机会，无需跳转到外部 BI 工具。

### 3.2 功能清单

| 编号 | 功能模块 | 优先级 | 描述 |
|------|---------|--------|------|
| FR-01 | 模式选择页（Mode Selector） | P1 | 新增独立路由页面（如 `/mode-select`），页面居中展示两张选项卡/按钮；左侧“openwork 原始版本”，右侧“harnesswork 工程驾驶舱”；选择原始版本直接跳转至当前首页，选择工程驾驶舱则跳转至 FR-02 页面 |
| FR-02 | 工程驾驶舱容器页（Cockpit） | P1 | 新增独立路由页面（如 `/cockpit`），整体采用上下分栏布局：上方固定 Tab 导航栏，下方为各 Tab 对应的内容区 |
| FR-03 | 驾驶舱 Tab 导航 | P1 | Tab 项包含：产品（Product）/ 研发（Engineering）/ 发布&运维（Release & Ops）/ 运营（Growth）；支持键盘导航；当前选中 Tab 高亮 |
| FR-04 | 产品 Tab 页 | P1 | 左侧显示产品文档树（遵循 Harness Engineering 文档体系，层级为 PRD / SDD / PLAN / TASK / MODULE），点击文档节点在右侧主区域展示文档内容（Markdown 渲染）；支持文档状态标记（draft/approved/released） |
| FR-05 | 研发 Tab 页 | P1 | 三栏布局：左侧代码文件树（支持折叠/展开、文件类型图标）、中间代码编辑区（语法高亮、行号、差异视图）、右侧 AI 对话区（参考 Qoder 布局，支持流式输出、工具调用展示、权限审批弹层） |
| FR-06 | 发布&运维 Tab 页 | P2 | 参考 Spinnaker / Harness CD / Argo CD 设计；包含：流水线执行视图（阶段/状态）、部署历史时间轴、多环境健康看板（dev/staging/prod）、告警摘要面板、快捷操作（审批/回滚） |
| FR-07 | 运营 Tab 页 | P2 | 参考 Amplitude / Mixpanel / PostHog 设计；包含：DAU/MAU 趋势图、用户留存热力图/漏斗、关键事件排行、用户反馈聚合列表（多渠道）、实验（A/B Test）结果摘要 |
| FR-08 | 模式记忆与快速切换 | P2 | 记住用户上次选择的模式（LocalStorage key: `harnesswork:mode-preference`）；应用冷启动默认进入 `/mode-select`，高亮上次所选选项，用户需主动点击确认进入对应模式（不自动跳过选择页）；在工程驾驶舱顶部提供返回模式选择入口 |
| FR-09 | 响应式适配 | P3 | 驾驶舱页面在大屏（≥1440px）和标准屏（≥1280px）下布局正常；研发 Tab 三栏在小屏（<1280px）下降级为双栏 |
| FR-10 | openwork 页面返回模式选择入口 | P2 | 在 openwork 原始版本（SessionView）左侧侧边栏顶部增加「← 返回模式选择」按鈕（data-testid: back-to-mode-select），点击后跳转至 `/mode-select`；与已有的工程驾驶舱返回按鈕行为保持一致 |
| FR-11 | 主题自适应 | P1 | 模式选择页（`/mode-select`）与工程驾驶舱（`/cockpit`）及其所有子组件跟随应用整体外观设置（亮色 / 暗色 / 跟随系统）自动切换；通过将硬编码深色 Tailwind 类替换为 DLS 语义 Token（`bg-dls-surface`、`text-gray-12`等）和 Radix 语义色阶（`bg-gray-4`、`text-green-11`等）实现；主题切换即时生效，无需刷新 |

### 3.3 非功能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 模式选择页首屏渲染 | < 500ms | 冷启动后模式选择页可见 |
| Tab 切换响应 | < 100ms | Tab 点击到内容区切换完成 |
| 文档树首次加载 | < 1s | 产品 Tab 文档树首次渲染 |
| 驾驶舱页面内存占用 | < 200MB | 四个 Tab 全部激活后总内存 |
| 代码编辑区流畅度 | 60fps 滚动 | 大文件（>1000行）滚动不卡顿 |

## 4. 验收标准

- [x] 应用启动后可路由到模式选择页（`/mode-select`），页面居中展示两个选项
- [x] 点击"openwork 原始版本"，跳转到当前首页（原 `/` 路由）
- [x] 点击"harnesswork 工程驾驶舱"，跳转到 `/cockpit` 页面
- [x] `/cockpit` 页面顶部显示 4 个 Tab（产品 / 研发 / 发布&运维 / 运营）
- [x] 点击产品 Tab，左侧显示 Harness Engineering 文档结构树，点击节点右侧渲染文档
- [x] 点击研发 Tab，呈现左（代码树）/ 中（编辑区）/ 右（AI 对话）三栏布局
- [x] 点击发布&运维 Tab，显示流水线视图、部署历史和环境健康看板
- [x] 点击运营 Tab，显示 DAU 趋势、留存漏斗和用户反馈模块
- [x] 用户模式选择记录到 LocalStorage，刷新后保留
- [x] 工程驾驶舱顶部提供返回模式选择入口
- [x] openwork 原始版本左侧侧边栏顶部提供返回模式选择入口
- [x] 模式选择页与工程驾驶舱在亮色 / 暗色 / 跟随系统三种主题下均可正常显示，文字可读，层次清晰
