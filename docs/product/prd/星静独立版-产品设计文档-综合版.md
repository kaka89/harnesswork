## 一、产品定位与愿景

### 1.1 核心定位

星静独立版是一款 **本地优先、AI 驱动的全流程产品研发协作平台**，面向创业者和小型团队。它内嵌于 harnesswork（OpenWork）桌面应用中，通过整合 OpenCode SDK、Tauri 原生能力和本地文件系统，为用户提供从产品规划到发布运维的完整工作流。

### 1.2 核心原则

| 原则 | 描述 |
|------|------|
| **本地优先** | 所有数据通过 OpenCode file API 或 localStorage 本地持久化，不依赖远程服务即可完整使用 |
| **多层降级** | OpenWork API → OpenCode SDK → Tauri fetch → localStorage，四级通道自适应，确保离线可用 |
| **信号驱动** | 基于 SolidJS `createSignal`/`createEffect` 实现细粒度响应式更新 |
| **模块化服务** | 每个业务领域独立 service 文件，通过函数导出解耦 |
| **OpenWork 复用优先** | 禁止重复建设 OpenWork 已有能力，优先集成 |

### 1.3 与团队版的差异

| 维度 | 独立版（Solo） | 团队版（Team） |
|------|--------------|--------------|
| 目标用户 | 创业者、独立开发者 | 工程团队（5-50人） |
| 数据存储 | Monorepo 单仓库 | 多仓库（Product Line/Domain/App） |
| 核心指标 | 商业指标（DAU/MRR/留存） | DORA 工程效能指标 |
| 发布流程 | 一键部署，无审批 | Tech Lead + SRE 双人审批 |
| Agent 集合 | 6 个 Solo Agent（产品脑/工程脑/增长脑/运维脑/数据脑/全能搭档） | 6 个 Team Agent（PM/架构/开发/QA/SRE/管理） |

## 四、核心功能模块详解

### 4.1 驾驶舱（Autopilot） — `pages/solo/autopilot/index.tsx`

**定位**：AI 虚拟团队主界面，对标 Claude Cowork

**核心能力**：

| 功能 | 描述 |
|------|------|
| 双模式切换 | 💬 普通对话（单 Agent Q&A）与 🚀 团队调度（多 Agent 并行） |
| EnhancedComposer | 自动高度输入框，支持 @mention Agent、/slash 命令、模型选择器 |
| SSE 流式会话 | 基于 OpenCode Session API 的实时消息流，支持心跳检测和断线重连 |
| 工具调用展示 | 实时显示 AI 工具调用过程（名称/参数/结果），支持权限审批 |
| 会话历史侧边栏 | 持久化多轮记忆，支持模式标识（对话/团队） |
| 产出物工作区 | 可 resize / 悬浮，支持保存到产品目录 |
| Agent 面板 | 可折叠侧边栏，显示各角色实时执行状态 |
| 知识注入 | 执行前自动检索相关知识（行为/私有/工作空间）注入上下文 |

**两阶段编排流程**：
```
用户输入目标 → Orchestrator 解析意图 → 生成 <DISPATCH> 计划 → 并发调用子 Agent → 汇总结果
```

**SSE 超时策略**：
- 首事件等待：60 秒（API Key 未配置检测）
- 活动心跳：120 秒
- 内容空闲：15 秒（兼容模型"思考后追加"场景）
- 断线重试退避：1s → 2s → 5s

```
  └─ 条件：isWorkspaceRelativePath(path) === true

Level 1: OpenCode SDK file API  
  └─ 本地 OpenCode 服务（默认 localhost:4096）
  └─ 条件：SDK client 可用且健康

Level 2: Tauri fetch 直连
  └─ 直接通过 Tauri HTTP 插件请求本地 OpenCode
  └─ 条件：Tauri 运行时可用

Level 3: localStorage 本地缓存
  └─ 浏览器环境最终兜底
  └─ 条件：所有上层通道失败
```

**通道偏好机制**：
- 首次加载从 Level 0 开始探测
- 成功后缓存偏好级别，后续直接使用
- 产品切换/工作目录变更时重置偏好
- **刷新按钮不重置偏好**（避免 10 路并发全部重新探测）
